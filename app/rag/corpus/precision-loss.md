# Precision loss

## Что это
Целочисленная арифметика Solidity округляет вниз. Неверный порядок операций
(divide before multiply), небезопасный downcast или потеря знаков приводят к
систематической ошибке: нулевые доли, заниженные выплаты, дрейф учёта.

## Как возникает
- Деление до умножения: `(a / b) * c` теряет остаток против `a * c / b`.
- Downcast `uint256 → uint128/uint64` без проверки диапазона.
- Округление в пользу протокола/пользователя без явного контроля.

## Признаки в коде
- divide before multiply в расчёте долей/наград;
- unsafe downcast (`uintN(x)`) без границы;
- rounding/decimal-математика без учёта остатка.

## Как чинить
- Умножать до деления; фиксировать направление округления осознанно.
- `SafeCast` (OpenZeppelin) для сужающих преобразований.
- Масштабирование (WAD/RAY) для дробной арифметики.

## Пример
```solidity
uint256 reward = (amount / totalSupply) * pool; // divide before multiply → округление в 0
uint128 packed = uint128(bigValue);             // unsafe downcast, тихое усечение
```
